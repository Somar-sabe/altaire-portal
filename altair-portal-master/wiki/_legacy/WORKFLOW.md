# Project Workflow

## Core Directives

**CRITICAL RULE:** We must work strictly **Module by Module** according to the defined arrangement. 
If the user deviates from the modular order or attempts to cross modular boundaries before completion, the AI must remind the user of this rule and redirect focus.

---

## 4-Phase Modular Execution Strategy

For each module in the agreed arrangement, we will follow these exact four phases before moving to the next module:

### Phase 1: Design & UI Foundation
- Finalize the design theme.
- Build unified, reusable components and templates specific to the module.
- Ensure strict adherence to Tailwind CSS and layout requirements.

### Phase 2: Data & Data Modeling
- Define the data schemas, interfaces, and mock structures.
- Connect the static data models to the UI components to establish visual completeness.

### Phase 3: Logic & Integration
- Implement operational business logic.
- Connect models and UI components to state management, APIs, and interaction handlers.

### Phase 4: Unit Testing & Debugging
- Thoroughly test the module's edge cases and interactions.
- Debug and optimize performance and reliability.
- Final review and sign-off before proceeding to the next module.
